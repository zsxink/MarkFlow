# MarkFlow 产品规格文档

> 版本：2.0.0 ｜ 状态：已发布 ｜ 更新日期：2026-07-02
> 平台：桌面端（macOS / Windows / Linux）
> 目标用户：写作者、开发者

---

## 1. 产品概述

### 1.1 产品定位

MarkFlow 是一款跨平台的所见即所得（WYSIWYG）Markdown 编辑器，面向写作者和开发者。它让用户在沉浸式的编辑环境中专注于写作，同时提供完整的 Markdown 语法支持。

**核心价值主张：**

- **所见即所得** — 输入即渲染，无需切换预览视图
- **真实文件系统** — 文件存储在用户选择的本地文件夹中
- **极简克制** — 干净无干扰的界面，让内容成为主角

### 1.2 目标用户

| 用户类型 | 使用场景 | 核心需求 |
| --- | --- | --- |
| 技术写作者 | 编写技术文档、API 文档 | GFM 完整支持、代码高亮、图片引用 |
| 内容创作者 | 写博客文章、笔记 | 所见即所得、主题美观、操作流畅 |
| 开发者 | 编写 README、项目文档 | 代码块语法高亮、Mermaid 图表 |
| 学生/研究者 | 笔记、论文草稿 | 简洁、专注、无干扰 |

### 1.3 产品原则

1. **极简主义** — 界面干净，工具栏按功能分组，专注模式隐藏所有干扰元素
2. **所见即所得** — Typora 式的编辑体验，输入即渲染
3. **本地优先** — 文件存储在用户本地的真实文件系统中，不强制云端同步
4. **高效导航** — 文件树 + 大纲双视图，快速定位文件和文档章节

### 1.4 v1 范围与里程碑

**v1 目标：核心功能可用**

- WYSIWYG Markdown 编辑（GFM 完整支持）
- 文件系统集成（打开文件夹、文件树）
- 主题系统（浅色/深色/护眼 Sepia）
- 设置面板
- 外部文件监听与冲突处理
- Mermaid 图表渲染

**v2 规划（不在 v1 范围）：**

- 用户自定义主题编辑器
- 主题市场/导入导出
- 导出 HTML/PDF
- 插件系统

### 2.2 用户交互流程

#### 2.2.1 首次启动流程

1. 用户启动 MarkFlow
2. 显示空编辑器（占位文字：「开始写作 — 输入即所得」）
3. 侧边栏默认展开，文件标签页显示空状态
4. 工具栏提示用户打开文件夹或新建文件
5. 读取配置文件，恢复主题等设置
6. 如果存在上次工作区，自动打开

#### 2.2.2 打开工作区流程

1. 用户点击「打开文件夹」或使用 Ctrl+O
2. 系统文件对话框打开（可选择文件夹）
3. 递归扫描文件夹中的所有 .md 文件
4. 构建文件树，填充侧边栏
5. 默认打开文件树中的第一个文件（如果有）
6. 记住该工作区路径

#### 2.2.3 编辑流程

1. 用户在所见即所得编辑器中输入
2. 实时解析 Markdown 语法，渲染为格式化内容
3. 状态栏实时更新字数、行数、光标位置
4. 自动保存定时器触发，保存到文件系统
5. 大纲视图自动更新（当前文档的标题层级）

#### 2.2.4 外部文件变化流程

1. Rust 端 notify 监听工作区文件变化
2. 检测到外部程序修改了当前打开的文件
3. 判断编辑器状态：
   - clean（无未保存修改）→ 自动重载文件内容
   - dirty（有未保存修改）→ 弹出冲突对话框
4. 忽略自身保存触发的文件事件（500ms 内）

## 3. UI/UX 规格

### 3.1 整体布局

采用 CSS Grid 三行两列布局：

```
┌─────────────────────────────────────┐
│             工具栏 (44px)            │
├───────────┬─────────────────────────┤
│  侧边栏    │                         │
│  (240px)  │       编辑区             │
│           │                         │
├───────────┴─────────────────────────┤
│           状态栏 (28px)              │
└─────────────────────────────────────┘
```

### 3.2 工具栏

规格详情见 `openspec/ui-design/SPEC.md`。

| 分组 | 按钮 |
| --- | --- |
| 文件操作 | 打开文件夹、新建文件、保存、设置 |
| 格式化 | 粗体、斜体、删除线、行内代码 |
| 插入 | 标题(H1/H2)、引用、链接、无序列表、有序列表、分隔线、代码块 |
| 模式切换 | 所见即所得、源码模式、专注模式 |
| 主题 | 主题切换按钮（最右侧） |

### 3.3 侧边栏

规格详情见 `openspec/ui-design/SPEC.md`。

- 宽度：240px
- 标签页：文件 / 大纲
- 文件树：支持文件夹嵌套、展开/折叠、拖拽移动、内联重命名、内联新建
- 大纲：自动提取 H1-H6 标题，层级缩进，点击跳转

### 3.4 编辑区

规格详情见 `openspec/ui-design/SPEC.md`。

- 最大宽度：720px
- 内边距：56px 48px 120px
- 所见即所得模式：Tiptap/ProseMirror GFM 渲染
- 源码模式：CodeMirror + 等宽字体
- 专注模式：隐藏工具栏、侧边栏、状态栏

### 3.5 状态栏

- 字数统计（按空格分词）
- 行数统计
- 光标位置（行 X, 列 Y）
- 当前模式指示器
- 快捷按钮：设置、专注模式、主题切换

### 3.6 设置面板

模态对话框，4 个标签页：

- **通用**：自动保存、拼写检查、软换行
- **外观**：主题预览、字号、行高
- **编辑器**：实时预览、代码高亮、行号、侧边栏默认展开、工具栏提示
- **快捷键**：格式化 + 视图快捷键参考

## 4. 验收标准

### 4.1 核心功能验收

| # | 用例 | 验收条件 |
| --- | --- | --- |
| 1 | 打开文件夹 | 点击「打开文件夹」→ 系统对话框打开 → 选择文件夹后文件树正确展示所有 .md 文件 |
| 2 | 创建文件 | 点击「新建文件」→ 文件树中出现内联输入框 → 输入文件名按 Enter → 文件出现在正确排序位置并自动打开 |
| 3 | 创建文件夹 | 点击「新建文件夹」→ 文件树中出现内联输入框 → 输入文件夹名按 Enter → 文件夹出现在正确排序位置 |
| 4 | 编辑文件 | 在编辑器输入 Markdown → 内容实时渲染 → 刷新后内容保留 |
| 5 | 保存文件 | Ctrl+S → 文件保存到文件系统 |
| 6 | 源码模式 | Ctrl+/ → 切换为 CodeMirror 纯文本模式 → 再次按 Ctrl+/ 切回 WYSIWYG |
| 7 | 专注模式 | Ctrl+Shift+F → 工具栏/侧边栏/状态栏隐藏 → 鼠标 hover 显示 → 再次按退出 |
| 8 | 外部重载 | 外部修改文件 → clean 状态自动重载，dirty 状态弹冲突对话框 |
| 9 | 主题切换 | 点击主题按钮 → 界面切换 → 刷新后保持 |
| 10 | 大纲视图 | 打开含标题文档 → 切换到「大纲」标签 → 显示 H1-H6 → 点击跳转 |
| 11 | 文件树操作 | 右键菜单（新建/重命名/复制/删除）→ 各操作正确执行 |
| 12 | 文件树拖拽 | 拖动文件到文件夹 → 移动到目标位置 |
| 13 | GFM 渲染 | 输入各种 GFM 语法 → 正确渲染 |
| 14 | Mermaid 图表 | 输入 Mermaid 代码块 → 渲染为 SVG |
| 15 | 设置持久化 | 调整字号 → 关闭 → 重新打开 → 字号保持 |
| 16 | 记住工作区 | 打开文件夹 A → 关闭 → 重新打开 → 自动打开 A |

### 4.2 非功能性验收

| 类别 | 标准 |
| --- | --- |
| 启动速度 | 冷启动 < 3 秒 |
| 编辑响应 | 打字延迟 < 50ms |
| 文件保存 | 保存 < 100ms（1MB 文件） |
| 内存占用 | 空闲时 < 200MB |
| 跨平台 | Windows/macOS/Linux 均可正常构建和运行 |
