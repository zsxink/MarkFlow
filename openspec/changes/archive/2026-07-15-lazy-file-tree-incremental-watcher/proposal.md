## Why

当前文件树会递归读取并序列化整个工作区，文件监听事件也会频繁触发整树重建，导致大型仓库启动缓慢、事件积压和界面状态抖动。需要将扫描、监听和渲染改为有界的增量流程，使 10k/100k 文件工作区以及依赖目录高频写入时仍保持可控的资源占用与响应速度。

## What Changes

- 后端提供浅层目录读取和分页/继续加载能力，前端仅在展开目录时加载子项，并限制单目录条目数量与递归深度。
- 文件树以规范化路径作为稳定 key，在创建、删除、修改和重命名事件到达时局部更新节点，避免单文件事件触发整树重建。
- watcher 对 `path + kind` 事件进行合并，通过有界队列施加 backpressure，并在溢出或状态不一致时执行受控重扫。
- 正确处理 notify 的重命名和批量事件；默认忽略 `.git`、`node_modules`、`target`、`dist` 等目录，并允许用户调整忽略规则。
- 记录扫描耗时、节点数量、事件队列长度和刷新次数，建立覆盖 10k/100k 文件、依赖目录高频写入、批量 Git 操作及极端目录结构的可重复基准。
- 对超大已加载节点集合评估并按基准结果引入虚拟列表，确保渲染成本有明确门槛。

## Capabilities

### New Capabilities

- `scalable-file-tree`: 定义文件树懒加载、增量事件处理、有界资源策略、可配置忽略规则、恢复机制、可观测性和性能基准。

### Modified Capabilities

无。

## Impact

- Rust/Tauri：目录读取命令、文件监听器、事件通道、忽略规则与性能指标。
- TypeScript/DOM：storage IPC 封装、文件树状态模型、懒加载渲染、增量 DOM 更新以及展开/选中状态维护。
- 设置：新增文件树忽略模式及加载上限的持久化配置，并提供安全默认值。
- 测试与基准：Rust watcher/扫描测试、前端增量 reducer/DOM 测试，以及大目录可重复基准工具。
- 现有 `read_dir_recursive` 调用将迁移到浅层读取接口；对用户可见的文件操作语义保持兼容。
