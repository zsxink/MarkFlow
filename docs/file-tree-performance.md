# 文件树性能预算

运行 `npm run benchmark:file-tree -- 10000` 或传入 `100000` 生成固定结构的大目录夹具并比较根目录浅层枚举与递归枚举。夹具同时包含 64 层深目录、单目录最多 5000 个文件，以及各 1000 个文件的 `node_modules`、`target` 高频写入替代场景；脚本还会建立 500 文件临时 Git 仓库并测量 branch checkout，以及合并 10k 原始事件。输出为 JSON，便于保存到 CI 工件做版本对比。

当前实现预算：单页 500 项、watcher 队列 2048 项、合并窗口 150ms、自动恢复展开深度 8。浅层首屏不得读取折叠目录；单文件事件不得增加根刷新计数；忽略目录事件不得进入队列。队列溢出只允许产生一个去重的最小受影响范围 `rescan` 恢复事件；无法确定范围时才退化为根恢复。

虚拟列表阈值暂定为 5000 个同时可见节点。懒加载和每目录分页使正常交互远低于该值，因此本次不引入虚拟化依赖；如果基准或运行指标显示可见 DOM 达到阈值，再以扁平可见行窗口化实现，并补键盘、拖拽和滚动定位测试。

## 100k 验收记录

固定种子 88 的本机验收结果：100,000 个分组文件、5,000 个单目录文件、64 层深目录及 2,000 个忽略目录写入文件。根目录浅层枚举 102 项耗时 0.96ms；递归枚举 105,165 个节点耗时 147.66ms。500 文件 Git branch checkout 为 1948.5ms；10,000 个原始修改事件合并为 500 个事件耗时 5.5ms。完整 JSON 可通过同一命令重复生成，具体耗时随磁盘与平台变化。
