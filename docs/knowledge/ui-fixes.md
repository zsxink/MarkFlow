# UI 修复经验

> 本文档记录历史排障背景和经验，**不是行为规范**。当前产品行为以 `openspec/specs/` 中对应模块规范和实现代码为准。

## 侧边栏布局

- CSS Grid 中使用随内容收缩的列，才能让折叠后的侧边栏不留下空白区域。
- 可折叠面板的最小宽度应由实际控件决定；拖拽范围与 CSS 最小宽度需要一致。
- 底部并列操作按钮适合使用 `flex: 1` 和 `white-space: nowrap`，避免宽度分配不均或换行。
- 图标状态变化需要由触发折叠/展开的事件同步更新。

## 编辑器统计

- ProseMirror 的 `textContent` 不保留块级节点之间的结构分隔，因此不能据此计算所见即所得模式的行数。
- 行数应遍历块级节点；字数仍可基于文本内容统计，并按需要处理 CJK 文本。

## 已过时的记录

早期记录曾建议源码编辑区固定为与所见即所得编辑器相同的 `1080px` 最大宽度和 `48px` 内边距。当前 CM6 源码编辑区使用独立布局，故该建议不再作为实现约束。
